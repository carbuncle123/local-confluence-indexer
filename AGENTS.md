# AGENTS.md

## 目的

このリポジトリは、Confluence のローカル同期と検索ツールを Python で実装するためのものです。
実装上の主な判断基準は [confluence_local_sync_index_spec.md](/Users/takeshi/ghq/github.com/carbuncle123/local-confluence-indexer/confluence_local_sync_index_spec.md) とし、挙動が曖昧な場合は独自解釈より先に仕様書を優先します。

## 作業方針

- Python 3.11 以上を前提にする。
- Python 依存管理は `uv` を使い、依存定義は `pyproject.toml` を正とする。
- `tools/` 配下には、役割が小さく明確なモジュールを置く。
- まず標準ライブラリを優先し、効果が大きい場合のみサードパーティ依存を追加する。
- 必要な箇所では型ヒントを付け、抽象化しすぎず追いやすい制御フローを選ぶ。
- `pathlib`、`argparse`、`sqlite3`、`json`、`logging` を素直に使う。
- 秘密情報はログ、標準出力、コミット対象、サンプル値に出さない。

## 実装の優先順

1. Confluence API クライアントと設定読み込みを作る。
2. SQLite によるローカル状態管理を整える。
3. retry と成功時のみの同期状態更新を含む sync フローを実装する。
4. frontmatter 付き Markdown 変換と安定したファイル名生成を実装する。
5. `manifest.jsonl`、`index.md`、ローカル FTS インデックスを構築する。
6. sync と search の CLI を使いやすい形で公開する。

## ファイルとデータの扱い

- 実装コードと設計を説明するドキュメントはコミット対象にする。
- Confluence から同期した本文、ローカル SQLite、`.env` はコミットしない。
- `docs/confluence/`、`.local-confluence-sync/`、`.local-doc-index/` は生成物として扱う。
- 新しいローカル生成物を増やした場合は、同じ作業内で `.gitignore` も更新する。

## Git 運用

- 実装作業を始めるときは、毎回必ず新しい作業ブランチを切る。
- ユーザー指定がなければ、ブランチ名は `codex/` プレフィックスにする。
- 1 つの論理的な変更に対応する、焦点の合ったコミットを作る。
- 実装はフェーズ単位で進め、各フェーズごとに PR を作成する。
- PR のタイトル、本文、コメントは日本語で書く。
- 次のフェーズへ進む前に、そのフェーズの処理内容と検証結果を確認できる状態にする。
- 明示的な依頼がない限り、ユーザーの変更を巻き戻したり書き換えたりしない。
- 大きめの編集の前後で `git status` を確認する。

## 検証

- まず最小限で意味のある検証から始め、必要に応じて広げる。
- まだ土台づくりの段階では、広い E2E より対象を絞ったスクリプト実行やテストを優先する。
- 作業完了前に、触ったコードの import、CLI 引数処理、パス解決を確認する。
- 各フェーズの完了時に、そのフェーズで追加した処理が問題ないかを確認してから次に進む。
- ローカルで確認できなかった点があれば、最後に明確に共有する。
